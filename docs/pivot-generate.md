# ピボット決定: 手作業CAD型 → AI生成型（2026-07-04）

## 背景（オーナーの言葉）

> 自分で色々と配置していくのはちょっと面倒で触らないなと思ってしまった。
> 0から作るとなったら考えるのも手を動かすのも面倒。

原因の診断: 間取りというテーマではなく**操作モデル**が問題。従来の間取りスタジオは「人間が0から手で配置するCAD型」で、ポートフォリオ全体の思想（人間の日課ゼロ・AIが動いて人間は承認だけ）と真逆だった。

## 新しい North Star

**「条件を選ぶ → 完成した部屋が複数案出てくる → 眺めて選んで、気になる所だけ直す」**

- 人間の仕事は「選ぶ」と「一言直す」だけ。0からの配置は人間の仕事ではない。
- 既存の3Dエディタは廃止しない。「生成された案の微調整ツール」に役割変更する。
- 既存資産（3Dビルダー・カタログ・実3Dサムネ・複数プラン保存）は全部生成型の土台として流用。

## アーキテクチャ

### レーン1: ルールベース生成（無料・即時・実装済みから始める）

- `src/lib/generate.ts` — Brief（間取りタイプ×暮らし×テイスト×seed）→ Plan JSON を決定的に生成
- テンプレート（1R / 1LDK / 2LDK / LUNAレッスンコース）＋壁沿い配置ルール＋テイスト別パレット
- seed違いで「3案ガチャ」。気に入らなければ引き直す
- APIコストゼロ、オフラインで動く、Vercel無料枠のまま

### レーン2: LLM生成（将来・要APIキー＝金銭コスト→オーナー承認待ち）

- 自然文（「小上がりのある1LDKで在宅ワークスペースがほしい」）→ Claude API が Plan JSON を出力
- Plan スキーマが単純（矩形の配列）なので LLM 生成と相性が良い
- 必要なもの: ANTHROPIC_API_KEY を Vercel に登録＋APIルート＋入力UI。Haiku なら1生成1円未満の見込み
- レーン1のテンプレを few-shot 例として使える

## 収益仮説（据え置き＋生成型で強化）

- 生成された案の家具に「これを買う」アフィリエイトリンク（楽天/Amazon、オーナーのASP登録待ち）
- 生成型はSEOと相性が良い: 「6畳 1K レイアウト 例」等の検索に対し生成例のギャラリーページを量産できる
